## 2. Pipeline Track B — kiến trúc

```
.tri source
    │
    ▼ triet-lexer + triet-parser       AST (arena-based, schema-generated types)
    ▼ triet-typecheck                  Vec<TypeError> — FATAL on error (blocking)
    ▼ triet-lower                      AST → MIR (flat, spanned, CallTarget-annotated)
    ▼ triet-mir                        Body(blocks, statements, terminators, CFG) + verify()
    ▼ triet-borrowck                   NLL dataflow → E24XX diagnostics (miette)
    ▼ triet-jit (mir_lower)            MIR → Cranelift IR → native x86-64 code
    ▼ execute                          i64 return → println!
```

### 2.1 — Đặc điểm kiến trúc

- **Schema-first:** `spec/schema/triet-schema.yaml` là single source of truth.
  Codegen (`codegen.py`) sinh Rust types trong `triet-syntax/src/generated/`.
- **MIR độc lập AST:** `triet-mir` là crate zero-dependency. Mọi type (Place,
  Local, Statement, Terminator, BinOp, CallTarget) tự định nghĩa.
- **Span xuyên suốt:** Mọi Statement và Terminator carry `Span = Range<usize>`
  từ AST. Diagnostics (E24XX) có line/column chính xác qua miette.
- **Borrowck NLL:** Polonius-style forward+backward dataflow. Field-level
  `places_conflict`. Cross-call `PropagatedLoan`. `miette::Diagnostic` derive.
- **JIT Bậc A:** Mọi giá trị là `i64` — scalars unboxed, aggregates là opaque
  pointer tới VM heap. Không native struct layout (Bậc C defer Phase 5).
- **Typecheck blocking:** Lỗi type = FATAL — pipeline dừng, không đẩy AST rác
  xuống lowerer/borrowck/JIT.
- **Type::Never:** Bottom type cho diverging blocks (có `return` statement).
  Thay thế `Type::Unknown` hack trước đây.
- **CallTarget enum:** `Jit` (Cranelift native) vs `Shim` (`extern "C"` SystemV).
- **Type-safe shim registry:** `ShimSymbol::fn_2_1()` factory methods — compile-time
  signature validation, không manual arity counting.
- **0 dead code:** Toàn bộ Track A (triet-ir, interpreter, bootstrap, cli,
  codegen.rs, shims.rs, aot.rs, loader.rs, legacy.rs) đã bị xóa vĩnh viễn.
- **func_id_for cache:** HashMap<String, FunctionId> đảm bảo same name → same ID
  trong MirBuilder session — API contract được tôn trọng.
- **DropWhileBorrowed (E2450):** Drop statement kiểm tra active loans trước khi
  retain — báo lỗi nếu biến đang được borrow, ngăn dangling reference.
- **StrongFrozen/StrongMutable conflict:** `conflicts_with()` trả về `true` cho
  strong forms — move luôn conflict với mọi active loan trên cùng place.
- **Outcome Bậc A pass-through:** Outcome ops là identity copy — Bậc C sẽ có
  packed representation với bit extraction thực sự.
- **LowerError thay panic:** `lower_program() -> Result<Vec<Body>, LowerError>`.
  0 `panic!()` trong toàn bộ lowerer. Mọi unsupported construct → `Err` với span.
  **(mentor review)**
- **places_conflict conservative:** Tham số `conservative: bool`. Khi `true`
  (BorrowReadOnly, WeakObserver, Move), 2 local khác nhau → assume conflict.
  Vá lỗ hổng alias analysis. **(mentor review)**
- **StructLayout từ AST:** `lower_program()` collect struct definitions, compute
  `StructLayout` với Bậc A sizes (8 bytes/field), attach vào mọi `Body`. Không
  còn dead code. **(mentor review)**
- **ReferenceForm → schema-generated:** Hand-written enum trong `type_ast.rs`
  được thay bằng `pub use crate::generated::types::ReferenceForm`. Type đầu
  tiên migrate từ schema sang compiler. **(mentor review)**
- **MIR verifier:** `Body::verify() -> Result<(), MirError>` — enforce INV-1
  (block bounds) + INV-2 (local bounds, incl. projection Index). Chạy sau lower,
  trước borrowck+JIT. 11 tests + regression guards. **(assignment #3)**
- **Outcome ops refuse:** JIT từ chối compile `OutcomeDiscriminant`/`OutcomeUnwrap`/
  `OutcomeUnwrapError` — pass-through identity là miscompile. Lowerer guard (0
  Outcome match) + JIT guard (refuse-over-guess). Test đỏ-khi-gỡ-guard.
  **(assignment #1)**
- **Multi-return refuse:** JIT từ chối `Return` với >1 value — trả 1 value là
  miscompile. Defer Bậc C packed ABI. **(assignment #2/#3)**
- **ConstValue truncate → Err:** `i128 → i64` ngoài range trả `JitError::Unsupported`,
  không `unwrap_or(0)` âm thầm. **(assignment #3)**
- **main() params refuse:** Driver từ chối gọi `main` có params — gọi sai arity
  đọc uninitialized Cranelift variables → UB. **(assignment #3)**

## 3. Chi tiết từng crate

### 3.1 — triet-mir (~1350 dòng, 15 tests)

**Vai trò:** Flat, non-nested IR với CFG. Independent crate, zero dependencies.

**Types chính:**
- `Local(usize)`, `BasicBlock(usize)`, `FunctionId(usize)`
- `Place { local, projection: Vec<Projection> }` — `Projection::Deref|Field(String)|Index(Local)`
- `Statement` (10 variants, tất cả carry `span: Span`)
- `Terminator` (5 variants: Return, Goto, If, CallDispatch, Unreachable — tất cả carry `span`)
- `Body { signature, blocks, entry_block, num_locals, local_decls, struct_layouts }`
- `ControlFlowGraph` — derived from Body via `build_cfg()` (predecessors/successors)
- `FunctionSignature { name, params, return_type, return_shape, return_borrow_map }`
- `BinOp` (20 variants): Add, Sub, Mul, Div, Mod, Eq, Ne, Lt, Le, Gt, Ge,
  LukAnd, LukOr, LukXor, LukImplies, LukIff, KleeneImplies, KleeneXor, KleeneIff, Neg
- `CallTarget { Jit, Shim }` — phân biệt ABI cho codegen
- `Span = Range<usize>`, `DUMMY_SPAN = 0..0` — cho synthetic nodes
- `ReturnShape { Unit, Scalar, BinaryOutcome, TernaryOutcome }`
- `Body::verify() -> Result<(), MirError>` — MIR verifier (INV-1 block bounds + INV-2 local bounds)
- `MirError { BlockOutOfBounds, LocalOutOfBounds }` — verification errors với span

### 3.2 — triet-lower (~750 dòng, 3 tests)

**Vai trò:** AST → MIR lowering. Threads AST spans vào MIR. **0 panic!()** —
mọi unsupported construct trả về `Err(LowerError)` với span.

**Public API:**
- `lower_program(&Program) -> Result<Vec<Body>, LowerError>` — mỗi `Item::Function` → 1 Body. Collect struct definitions → `StructLayout`, attach vào mọi body.
- `lower_function(&FunctionDef, &Arena, Span) -> Result<Body, LowerError>`

**Hỗ trợ:**
- Statements: let, expression, return, assignment, while
- Expressions: integer/trit/trilean literals, identifier, binaryop (18/19 operators),
  call, block, if/else, borrow (&0/&+/&-), field access (→ Place projection)
- **Pow operator:** lowered to `CallDispatch(target=Shim, callee="__triet_pow")`
  — không fallback Assign. Hỗ trợ nested expressions: `a + b ** c`, `2**10 + 3**5`
- **StructLayout populate:** `lower_program()` duyệt `Item::Struct`, compute layout
  với Bậc A sizes (8 bytes/field, align 8), gán vào `Body::struct_layouts`.
- DUMMY_SPAN cho synthetic code (implicit Return, while-loop Goto)

**Error handling:** `LowerError` struct với 4 factory methods — `unsupported_stmt`,
`unsupported_expr`, `unsupported_callee`, `undefined_local`. Tất cả carry `Span`.
Driver hiển thị lỗi + exit code 3.

**Chưa hỗ trợ:** String/Vector/HashMap/Enum/Struct literals (trả về `Err`, không panic).

### 3.3 — triet-borrowck (~980 dòng, 10 tests)

**Vai trò:** NLL borrow checking trên CFG. Forward dataflow + backward liveness.

**Public API:**
- `checker::check_body(&Body) -> BorrowCheckResult`
- `checker::check_body_with(&Body, &callee_sigs) -> BorrowCheckResult`

**Error types (miette::Diagnostic + thiserror::Error):**
- `E2420 UseAfterMove { local, name, span }` — `#[label("used here after move")]`
- `E2440 NllExclusivityViolation { source_local, source_name, new_form, existing_loan_dest, span }` — `#[label("conflicting borrow created here")]`
- `E2450 DropWhileBorrowed { local, name, span }` — `#[label("drop occurs here while borrows are still active")]` **(added post-audit)**

**Thuật toán:**
- `places_conflict(a, b, conservative)`: **(updated — mentor review)**
  - Khi `conservative=true`: 2 local khác nhau → assume conflict. Dùng cho `BorrowReadOnly`, `WeakObserver`, và Move — những form cho phép aliasing.
  - Khi `conservative=false`: 2 local khác nhau → assume disjoint (exclusive guarantee của `BorrowExclusiveMutable` và strong forms).
  - Cùng local: field-level comparison như cũ — `obj.x` vs `obj.y` → disjoint.
- `Loan { source: Place, dest: Local, form, issued_in, issued_at }`
  - `conflicts_with(new_form)`: StrongFrozen/StrongMutable → `true` (move conflicts with ALL loans) **(fixed post-audit)**
- `BlockState { var_states, active_loans }` — merge conservative (Moved if ANY path)
- Fixed-point iteration trên CFG worklist
- Cross-call `PropagatedLoan` qua `return_borrow_map`
- Terminator-level errors dùng terminator's own span (không DUMMY_SPAN)
- `func_id_for()`: HashMap cache — same name → same FunctionId **(fixed post-audit)**

### 3.4 — triet-typecheck (fixes)

**Vai trò:** Type validation — BLOCKING. Errors abort pipeline.

**Fixes trong phiên này:**
- `check_block`: block có `return` statement → `Type::Never` (bottom type).
  Individual `return` statements đã được validate riêng với declared return type.
- `Type::Never` thêm vào `matches()`: compatible với mọi type (universal subtype).
- `is_concrete()`: `Never` treated as non-concrete (giống `Unknown`).
- `Display`: `Never` → `"!"`.

### 3.5 — triet-jit (mir_lower.rs ~1150 dòng, 15 tests)

**Vai trò:** MIR → Cranelift IR → native x86-64 code. Active JIT path (Track B).

**Public API:**
- `JitContext::new()` / `JitContext::with_shims(&[ShimSymbol])`
- `compile(&Body) -> Result<CompiledFunction, JitError>`
- `compile_multi(&[&Body]) -> Result<HashMap<String, CompiledFunction>, JitError>`

**CompiledFunction API:**
- `call_i64_0() -> i64`
- `call_i64_1(i64) -> i64`
- `call_i64_2(i64, i64) -> i64`

**Cranelift lowering (lower_binop, 140 dòng):**
- **Arithmetic:** Add/Sub/Mul → `iadd`/`isub`/`imul`, Div/Mod → `sdiv`/`srem`, Neg → `ineg`
- **Comparisons (→ Trilean!):** `icmp` + `select(+1, -1)` — không bao giờ Unknown
- **Łukasiewicz Ł3:** And=min, Or=max (qua icmp slt/sgt + select)
  - Implies: chain 6 lệnh Cranelift — a=False→True, a=True→b, a=Unknown→(b==False?0:+1)
  - Iff/Xor: dùng lại implies + and/neg
- **Kleene K3:** Implies = max(-a, b)
- **Reference:** `triet-logic/src/trilean.rs` cho tất cả truth tables

**SSA handling:** Cranelift `FunctionBuilder` + `Variable` — KHÔNG tự viết SSA pass.

**CFG traversal:** Reverse Post-Order (RPO). Block sealing khi tất cả predecessors đã fill.

**Shim call infrastructure:**
- `ShimSymbol` struct với type-safe factory methods: `fn_0_1()`, `fn_1_1()`, `fn_2_1()`, `fn_2_0()`.
  Mỗi method nhận `extern "C" fn` với signature cụ thể → compile-time arity validation.
- `CallTarget::Shim` → `Linkage::Import` + `CallConv::SystemV`
- Tất cả shim functions có `#[allow(unsafe_code)]` + `#[unsafe(no_mangle)]` + `extern "C"`
- Test: `__test_shim_multiply(7, 9) = 63` qua JIT
- Test: `__triet_pow(2, 10) = 1024` qua JIT

**Bậc A — single i64 ABI:**
- Mỗi MIR `Local` → 1 Cranelift `Variable` (không split disc/payload)
- **Outcome ops: pass-through identity** (copy i64 unchanged). Real bit-extraction
  (Trit discriminant masking, payload shifting) deferred to Bậc C packed ABI.
  Documented in `mir_lower.rs` with expanded test doc-strings. **(doc added post-audit)**
- Function return: luôn 1 i64 (multi-value return deferred to Bậc C)

**Tests (15):**
- 5 arithmetic/control flow: abs_diff, add, fibonacci (recursive!), outcome_discriminant, outcome_caller_callee
- 8 logic truth tables: LukAnd, LukOr, LukImplies (9 combos), LukIff, LukXor, KleeneImplies, LukVsKleene, Neg
- 2 shim calls: multiply, pow

### 3.6 — triet-driver (~130 dòng)

**Vai trò:** Binary pipeline driver cho Track B.

**Usage:**
```bash
triet-driver <file.tri>         # check mode: parse→typecheck→lower→borrowck
triet-driver run <file.tri>     # run mode: check + JIT compile + execute
```

**Pipeline:**
1. `triet_parser::parse(&source)` → Program
2. `triet_typecheck::check(&program)` → **FATAL on error** (return ExitCode::from(3))
3. `triet_lower::lower_program(&program)` → `Result<Vec<Body>, LowerError>`
4. **`body.verify()` trên từng Body** → `Err(MirError)` nếu malformed MIR (INV-1, INV-2)
5. `triet_borrowck::checker::check_body(body)` → borrow errors (miette, blocking)
6. `JitContext::with_shims(&[ShimSymbol::fn_2_1("__triet_pow", ...)])` → JIT compile
7. Find `main`, check params empty → `call_i64_0()` → print result

## 4. Các quyết định kiến trúc quan trọng

### 4.1 — Span trong MIR

Mọi Statement và Terminator carry `span: Span`. Lowerer threads AST `Spanned<T>.span`.
`DUMMY_SPAN` CHỈ cho synthetic code (implicit Return, while-loop Goto, dead blocks).
Terminator-level errors dùng terminator's own span — không DUMMY_SPAN.

### 4.2 — Logic ops = icmp/select chains, không bitwise

Trilean encoding: +1=True, 0=Unknown, -1=False. Logic ops reference từ
`triet-logic/src/trilean.rs`. Tất cả 66 assertions truth table pass.

### 4.3 — Shim calls = extern "C" + SystemV ABI + `#[no_mangle]`

Rust ABI không ổn định — không được phép giả định tương thích SystemV.
Tất cả shim functions: `#[allow(unsafe_code)]` + `#[unsafe(no_mangle)]` + `extern "C"`.
Cranelift dùng `CallConv::SystemV` cho `Linkage::Import`.
Type-safe registration: `ShimSymbol::fn_2_1("name", function_ptr)` — compile-time
signature check, không manual arity counting.

### 4.4 — Bậc A = mọi thứ là i64

Scalars unboxed. Aggregates (struct, enum, outcome, string, vector, hashmap) =
opaque i64 pointer → delegate-to-VM shim. Outcome ABI là 1 i64.
Native struct layout (Bậc C) defer Phase 5.

### 4.5 — Type::Never (bottom type)

Diverging blocks (chứa `return`/`break`/`continue`) có type `Never`.
`Never.matches(X)` = true (bottom type, compatible với mọi type).
Thay thế `Type::Unknown` hack trước đây — chính xác về mặt type theory.

### 4.6 — Pow = shim call, không fallback Assign

`Expr::BinaryOp(Pow)` lowered thành `CallDispatch(target=Shim, callee="__triet_pow")`.
Hỗ trợ nested expressions: `a + b ** c`, `2**10 + 3**5`.
Cranelift `Variable` track xuyên block → không cần block-split đặc biệt.

## 8. Bài học

### Từ Phase 2 & 3 implementation

1. **Cross-crate impact audit:** Place vs Local type change gây ~20 vị trí
   cần sửa trong mir_lower.rs — phát hiện muộn lúc compile, không có trong plan.
2. **`extern "C"` là contract duy nhất:** Giả định "Rust ABI = SystemV" là UB.
   Mentor bắt lỗi — sửa với `#[no_mangle]` + type-safe factory methods.
3. **DUMMY_SPAN kỷ luật:** Chỉ dùng cho synthetic code. Terminator errors
   dùng terminator's own span.
4. **Test truth table trước khi claim xong:** 66 assertions cho Ł3/K3 logic ops
   là cứu cánh.
5. **Defer là nợ máu:** Outcome 2-register ABI bị defer qua 2 phase — đã tháo.
6. **`Type::Unknown` hack cho diverging block là sai:** `Type::Never` (bottom type)
   mới là câu trả lời đúng về type theory.
7. **Dead code phải xóa, không feature-gate:** 5500 dòng legacy đã bị xóa vĩnh viễn.
   Git history giữ lại nếu cần — không cần `#[cfg(feature = "legacy")]`.
8. **Typecheck phải blocking:** Đẩy AST lỗi xuống lowerer/borrowck/JIT là bắt
   các layer dưới ăn rác.

### Từ self-audit

9. **Green tests ≠ correct code:** 262 tests pass nhưng self-audit phát hiện 2
   soundness holes (Drop-while-borrowed, StrongFrozen conflict), 1 API bug
   (func_id_for), và Outcome pass-through chưa được document. Không audit =
   không biết code mình sai.
10. **Schema ≠ reality:** Generated `Type` enum 24 variants là dead code —
    typechecker dùng hand-written Type riêng. "Single source of truth" mới chỉ
    là aspiration, chưa phải reality.
11. **StructLayout là dead code có tests:** 4 tests cho algorithm KHÔNG được
    dùng trong production. Tests cho dead code vẫn pass — phản tác dụng.
12. **API contract trong doc-comment phải được implement:** `func_id_for` document
    nói "Returns the same ID for the same name" nhưng code luôn tạo ID mới.
    Doc comment sai = bug, không phải "chưa implement."
13. **Borrowck cần DropWhileBorrowed:** Drop một biến đang được borrow → reference
    thành dangling. Đây là lỗ hổng cơ bản mà test suite không phát hiện vì
    không có test case Drop interaction.
14. **Strong forms là move, phải conflict:** `&+ obj` (StrongFrozen) move obj —
    phải conflict với mọi active loan trên obj. `conflicts_with` trả về `false`
    cho strong forms là sai cơ bản.
15. **32 Track B tests là quá ít:** Pipeline 7 tầng (parse→typecheck→lower→MIR→
    borrowck→JIT→execute) nhưng chỉ 32 tests. 4 trong số đó test dead code.
    15 JIT tests chủ yếu test logic ops. Hầu như không có test cho error paths.

### Từ mentor review

16. **Compiler không được panic:** `panic!()` trong lowerer là lười biếng. User input
    không bao giờ được crash compiler. `Result<Vec<Body>, LowerError>` là baseline,
    không phải feature.
17. **Schema phải chứng minh giá trị:** Generated `Type` enum 24 variants là dead
    code — nhưng schema-generated `ReferenceForm` đã được migrate thành công vào
    typechecker. Chứng minh pipeline schema→codegen→compiler hoạt động.
18. **Borrowck không alias = không borrowck:** `places_conflict` giả định "khác
    local = disjoint" là soundness hole cho shared/weak references. Conservative
    assumption (`conservative=true`) là band-aid — cần alias analysis tử tế.
19. **StructLayout phải sống:** Dead code trong MIR là technical debt. Populate
    từ AST ngay bây giờ, dù Bậc A chỉ có 8-byte fields. Khi Bậc C đến, chỉ cần
    thay sizes — không cần thay architecture.
20. **Mentor không quan tâm đến "green tests":** 0 warnings, 0 errors, all green
    là điều kiện cần, không phải điều kiện đủ. Mentor nhìn vào architecture
    soundness, không phải test suite color.

### Từ assignments #1-#3

21. **Test đỏ-khi-gỡ-guard mới là test thật:** `outcome_discriminant_jit_refuses_to_compile`
    không test "code hoạt động" — nó test "guard còn sống." Nếu ai gỡ guard, test PANIC
    với message chính xác. Đây là pattern cho mọi latent guard trong compiler.
22. **Bản đồ trước khi sửa:** 18 điểm panic/UB → 2 invariant (block bounds + local bounds).
    Không vá 18 chỗ — dựng 1 verifier. Whack-a-mole là phản pattern; invariant
    synthesis mới là engineering.
23. **Verifier là boundary, không phải feature:** `Body::verify()` chạy MỘT LẦN sau
    lower, trước borrowck+JIT. Mọi consumer phía sau được phép assume MIR well-formed.
    Một verifier không phủ kín (sót projection Index) tệ hơn không có verifier.
24. **Refuse over guess áp dụng cho cả compiler internals:** Outcome ops → refuse.
    Multi-return → refuse. ConstValue truncate → refuse. main() params → refuse.
    "Thà từ chối compile còn hơn sinh code sai" là nguyên lý xuyên suốt.
25. **AOT cache là premature optimization:** Compiler compile mili-giây, chưa có
    core semantics đầy đủ. Xây mái khi móng còn nứt là sai. Mentor kill đúng.
