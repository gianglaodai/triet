### 1.2 — Code generator (HOÀN THIỆN)

**`spec/schema/codegen.py`** sinh 5 file Rust từ schema:

```
crates/triet-syntax/src/generated/
├── mod.rs           — module index + re-exports
├── types.rs         — PrimitiveType, ReferenceForm, Type (30 variants), visitor + accept
├── ast_operator.rs  — BinaryOperator (20 variants), UnaryOperator (3 variants)
├── ast_expr.rs      — Expr (32 variants)
├── ast_stmt.rs      — Stmt (10 variants)
└── ast_item.rs      — Item, Program, FunctionDef, StructDef, EnumDef, etc.
```

Các tính năng codegen đã implement:
- Parse YAML schema (đã chuyển từ JSON sang YAML để hỗ trợ comment)
- Sinh `enum` + `struct` Rust với derive macros (Clone, Debug, PartialEq, Eq)
- Sinh visitor trait (`TypeVisitor`) + `accept()` method cho Type enum
- Builtin type resolution: tự động map schema type names → Rust type paths
- Xử lý recursive types: tự động bọc `Box<T>` cho field tham chiếu đến enum
- Phát hiện và tránh `Box<Vec<T>>`, `Box<Option<T>>` dư thừa
- Schema validation (`--validate`) — kiểm tra completeness và consistency

## 3. Parser migration — hành trình và bài học

### 3.1 — Những gì đã thử

Đã thử big-bang migration parser → generated types **10+ lần** với 3 cách:

| Cách | Kết quả | Lỗi điển hình |
|---|---|---|
| `sed` regex toàn workspace | Bracket mismatch | Nested parens (`Vec::new()`) bị cắt |
| Python script với regex | Double-renames | `params:` → `params: params:`, Block keyword bị xóa |
| Sửa tay (`cargo check` → fix → repeat) | 103 → 2 lỗi | 30s/vòng feedback loop, lỗi mới xuất hiện khi fix lỗi cũ |

### 3.2 — Tiến bộ thực tế

Dù không hoàn thành, đã đạt được tiến bộ đáng kể:

- **BinaryOperator renames** (16 variants, ~100 vị trí): Hoàn thành qua IDE và sed
- **ParameterPassing renames**: Hoàn thành
- **Field renames** (parameters→params, passing→passing_mode, etc.): Hoàn thành
- **Expr tuple→struct variants**: Phần lớn hoàn thành
- **Block type removal + parse_block refactor**: Hoàn thành — đây là phần phức tạp nhất
- **Item::Function/Struct/Enum tuple→struct**: Hoàn thành
- **Program construction**: Hoàn thành

Đã có thời điểm chỉ còn **2 lỗi** trong parser.

### 3.3 — Vấn đề gốc rễ

Generated types có **225 vị trí khác biệt** với old types:

| Loại khác biệt | Số vị trí |
|---|---|
| Tuple → Struct variant (15 Expr variants) | ~50 |
| Field rename (20 tên field) | ~30 |
| Block type removal (inlined vào Expr::Block) | ~15 |
| BinaryOperator variant names (16 variants) | ~100 |
| Khác (ParameterPassing, Item variants, etc.) | ~30 |

Trong đó, **3 khác biệt là semantic**, không phải mechanical rename:

| Vấn đề | Parser dùng | Generated dùng |
|---|---|---|
| `StructField` field type | `TypeId` (arena handle) | `Type` (resolved type) |
| `For.variable` type | `PatternId` | `String` |
| `Assignment.target` type | `String` (identifier name) | `ExprId` |

Những khác biệt semantic này đòi hỏi thay đổi logic parser hoặc schema — không thể fix
bằng text replacement.

### 3.4 — Cascade effect

Khi wire generated types vào `lib.rs` re-exports, lỗi lan sang các crate khác:
- `triet-modules` (23 lỗi)
- `triet-interpreter` (35 lỗi)

Điều này xác nhận mentor nói đúng: "không xây nhà từ nóc" — phải thông toàn bộ
pipeline từ parser → typecheck → lowerer trước khi đụng đến borrow checker.

### 3.5 — Kết luận

1. **Big-bang migration qua CLI agent là không khả thi.** Cần IDE với real-time feedback.
2. **Generated types có 3 semantic mismatch với parser** — cần sửa schema HOẶC parser logic.
3. **225 vị trí mechanical** — làm được trên IDE trong ~40 phút.
4. **Cần migrate TOÀN BỘ pipeline** (parser + typecheck + lowerer) cùng lúc, không từng crate một.
